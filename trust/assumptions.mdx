## The largest assumption: the oracle

<Warning>
During early phases, **LPs trust Hyro's oracle infrastructure** to accurately report off-chain balances, trading performance, and exchange-held capital. While settlement, share accounting, fee logic, and payouts all occur on-chain, NAV reporting currently depends on oracle infrastructure operated by Hyro. This is the single largest trust assumption in Phase 1, and we disclose it explicitly.
</Warning>

**The path to removing it:**

| Stage | Mitigation |
|---|---|
| Today | Independent watchdog cross-validation |
| Phase 2 | Multi-signer oracle |
| Long-term | ZK proofs of CEX state via zkTLS |

## Decentralization, stage by stage

| Layer | Today | Roadmap |
|---|---|---|
| Capital settlement | On-chain (Solana) | — |
| Share accounting | On-chain | — |
| LP deposits / withdrawals | On-chain | — |
| Performance-fee logic | On-chain | — |
| Payout records | On-chain (TXID each) | All future payouts on-chain by default |
| NAV reporting | Hyro oracle + watchdog | Multi-signer (Phase 2); zkTLS (long-term) |
| Trader admission | HyroTrader pipeline | Permissionless challenges (Phase 3) |
| Execution venue | Bybit | Multi-CEX; Drift + Phoenix on-chain (Phase 4) |
| Operator wallet | Squads 3/5 multisig | DAO-controlled (Phase 4) |
| Risk parameters | Hardcoded (mirror HyroTrader) | DAO-governed (Phase 3) |
| Governance | Multisig admin | $HYRO token governance (Phase 3) |

<Info>
We are progressively decentralizing. Stage 1 today: decentralized capital, transparent on-chain settlement, programmatic rules. Stage 2: permissionless trader onboarding. Stage 3: on-chain execution. Stage 4: own execution stack, removing all third-party dependencies.
</Info>
